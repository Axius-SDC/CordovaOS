# CordovaOS

[![License](https://img.shields.io/badge/License-Apache_2.0-blue.svg)](LICENSE)
[![SDC4](https://img.shields.io/badge/SDC4-Compliant-teal.svg)](https://semanticdatacharter.com)

**A fictional nation's entire government — 10 domains, zero integration code.**

CordovaOS is a proof-of-concept demonstrating how [SDC4](https://semanticdatacharter.com) (Semantic Data Charter) enables deterministic cross-domain interoperability. It models the Republic of Cordova, a fictional island nation, with 10 fully operational government domain applications that share data through a common semantic foundation — no middleware, no API adapters, no ETL pipelines, no data mapping layer.

## Live Demo

[CordovaOS Demo](https://cordovaos-web-43199315497.us-central1.run.app/demo/)

## Architecture

All 10 domain apps were generated by [SDCStudio](https://sdcstudio.axius-sdc.com/) AppGen from SDC4-compliant data models. They interoperate because every app shares the same SDC4 Reference Model.

| Domain | Description |
|--------|-------------|
| **Civil Registry** | Citizen identity records — National IDs (CIDs) shared across all domains |
| **Vital Statistics** | Birth, marriage, and death certificates |
| **Business Registry** | Business entity registrations with Business Registry Numbers (BRNs) |
| **Property Registry** | Land and property ownership records |
| **Education Record** | Student enrollments, faculty appointments, academic records |
| **Employment Record** | Employment history linking persons to businesses |
| **Tax & Revenue** | Tax filings for individuals and businesses |
| **Maritime Port Authority** | Vessel port calls, crew manifests, cargo manifests |
| **Healthcare Record** | Patient encounters, vitals, lab results, diagnoses |
| **Law Enforcement Record** | Incident reports and case files |

## The Contagion

The centerpiece of CordovaOS is a 7-beat crisis narrative that demonstrates cross-domain graph traversal in action:

1. **The Arrival** — A cargo vessel docks at Porto Sereno. A crew member falls ill.
2. **The Diagnosis** — The hospital identifies a novel pathogen.
3. **The Trace** — Contact tracing crosses maritime, civil, employment, and education domains.
4. **The Spread** — University exposure expands the contact network to 356 people across 4 tiers.
5. **The Response** — The Provincial Health Officer triggers cross-domain containment.
6. **The Investigation** — Law enforcement opens an incident tied to maritime and health records.
7. **The Resolution** — A complete government profile query shows every touchpoint for the index case.

Every beat is answerable by SPARQL queries against the shared knowledge graph — no custom integration required.

- **Django 5.2** — Web framework for all domain applications
- **PostgreSQL** — Operational database
- **GraphDB** — OWL 2 RL reasoning triplestore for RDF/SPARQL
- **SirixDB** — Temporal JSON/XML versioning database
- **Keycloak** — Enterprise SSO (OAuth2, OIDC, SAML 2.0)
- **Docker / Podman** — Containerized deployment
- **Google Cloud Run** — Production hosting

## Quick Start

```bash
git clone https://github.com/Axius-SDC/CordovaOS.git
cd CordovaOS/app/sdc4
cp .env.example .env
docker compose up -d --build
```

Access the application at http://localhost:18000 (default credentials: `admin` / `admin123`).

See [app/sdc4/README.md](app/sdc4/README.md) for detailed setup, port configuration, and production deployment instructions.

> **Memory**: The Enterprise Stack requires 6GB+ free RAM for GraphDB, SirixDB, Keycloak, PostgreSQL, and Redis.

## SPARQL Queries

Seven pre-built queries in `sparql/` demonstrate deterministic cross-domain graph traversal:

| # | Query | Domains Traversed |
|---|-------|-------------------|
| 1 | Complete Government Profile | All 10 |
| 2 | Economic Network Analysis | Employment, Business, Tax, Maritime, Property |
| 3 | Social Services Identification | Healthcare, Civil Registry, Employment, Property |
| 4 | Supply Chain Provenance | Maritime, Business, Tax |
| 5 | Family Economic Unit | Civil Registry, Employment, Property, Tax, Education |
| 6 | Institutional Impact | Education, Employment, Business, Tax |
| 7 | Contagion Contact Tracing | Maritime, Civil Registry, Employment, Education |

Cross-domain joins work through two mechanisms:
- **Shared component ct_id** — National ID and Business Registry Number components are reused by ct_id across domain schemas
- **Party identity** — `sdc4:party-id` carries CID (persons) or BRN (businesses) for cross-domain identity resolution

## How It's Built

Each domain app is generated by [SDCStudio](https://sdcstudio.axius-sdc.com/) AppGen:

1. **Model** the domain in SDCStudio using SDC4-compliant components
2. **Generate** a complete Django app with XSD, RDF/OWL, SHACL, and JSON-LD schemas
3. **Deploy** via the SDC4 project container (one `docker compose up` for all domains)
4. **Import** synthetic data as XML instances validated against each domain's XSD

The apps interoperate with zero glue code because SDC4 schemas share a common reference model. Components reused across domains (like National ID) share the same `ct_id`, producing identical RDF predicates that join naturally in SPARQL.
